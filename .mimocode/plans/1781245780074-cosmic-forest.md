# API Deep Research 機能追加計画

## 概要

検索画面を2タブ構成に変更し、API経由でDeep Researchを実行する方式を追加する。

## UI構成

```
検索画面
  ├─ Step 1: 論文要約（共通、変更なし）
  ├─ Step 2: ジャーナル検索
  │   ├─ タブ1: APIでDeep Research（新規）
  │   └─ タブ2: 外部Deep Researchを利用（既存貼り付け方式を移動）
  └─ 候補ジャーナル表（共通、変更なし）
```

### タブ1: APIでDeep Research

```
Deep Research プロバイダー選択:
  - Perplexity Sonar Deep Research
  - OpenAI Deep Research
  - Gemini Deep Research Agent
  - Claude Web Search
  - Custom Search + LLM（将来）

調査方針:
  - 低APC優先
  - マッチ度優先
  - バランス
  - 高IFチャレンジ候補も含める

検索対象指示:
  - ジャーナル公式サイト優先
  - PubMed / Crossref / DOAJ / Scimago 含める
  - predatory journal チェック含める

[Deep Research を実行] ボタン

結果: 候補ジャーナル表（既存のJournalCandidateに統合）
```

### タブ2: 外部Deep Researchを利用（既存）

```
プロンプト生成 → コピー → 外部AIに貼り付け → 結果貼り戻し → 解析
（現在の実装をそのまま移動）
```

## 対応API（実装順）

### Phase 1: UIのみ（今回実装）
- 検索画面を2タブに分割
- タブ1にProvider選択UIを配置（実API呼び出しなし）
- 既存貼り付け方式をタブ2に移動
- 既存機能は壊さない

### Phase 2: Perplexity Sonar Deep Research
- `PERPLEXITY_API_KEY` を使用
- `sonar-deep-research` モデル
- Web検索・引用付き回答
- JournalCandidate への変換

### Phase 3: OpenAI Deep Research
- `o3-deep-research` / `o4-mini-deep-research`
- Responses API（chat/completions とは別）
- `OPENAI_API_KEY` を使用

### Phase 4: Gemini Deep Research Agent
- Gemini API Deep Research Agent
- Gemini Grounding with Google Search（別モード）
- `GEMINI_API_KEY` / `GOOGLE_API_KEY`

### Phase 5: Claude Web Search
- Claude API Web Search tool
- `ANTHROPIC_API_KEY`
- コスト高め、確認用・高品質版として位置づけ

### Phase 6: Custom Search + LLM（将来）
- Search API + LLM による自作Research Pipeline

## Rust側モジュール構成（将来見据え）

```
core/
  deep_research.rs        # 既存（貼り付け方式パース）
  deep_research_api.rs    # 新規（API方式共通インターフェース）
  deep_research/
    mod.rs
    perplexity.rs         # Perplexity Sonar API
    openai_dr.rs          # OpenAI Deep Research API
    gemini_dr.rs          # Gemini Deep Research Agent
    claude_ws.rs          # Claude Web Search
```

## 実装方針

- JournalCandidate 構造体は既存のまま（拡張は後で）
- 引用・根拠リンクは `source_evidence` フィールドに保存
- コスト警告: Claude Web Search の高コストをUIに表示
- エラー表示: URL / HTTP status / レスポンス詳細
- 既存の貼り付け方式は壊さない

## 制限事項

- Python は使わない
- Rust + Tauri + React で実装
- まず計画のみ、実装は Phase 1（UI）から段階的に
